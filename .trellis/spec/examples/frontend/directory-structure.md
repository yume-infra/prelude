# Directory Structure

> How frontend examples are organized.

---

## Overview

`apps/examples` does not contain hand-written frontend source. Frontend examples are generated during linked smoke tests into `apps/examples/.generated/`.

The source of truth for those projects is the CLI template system under `apps/cli/templates/` and `apps/cli/src/core/template-registry/`.

## Directory Layout

```text
apps/examples/
  README.md
  .generated/
    react-full-linked/   generated by linked smoke
    vue-full-linked/     generated by linked smoke
```

The `.generated/` tree is ignored by git and may be removed/recreated by smoke tests.

## Module Organization

- Generated React/Vue app files are not maintained in this package.
- Any change to example frontend output should be made in CLI templates, owners, schemas, or registry code.
- Linked smoke validates the generated output as user-visible product behavior.

## Examples

- `apps/cli/tests/linked-examples.smoke.ts` generates `react-full-linked` and `vue-full-linked` under `apps/examples/.generated`.
- `apps/cli/templates/fragments/react/App.tsx.hbs` is the source for generated React app shell output.
- `apps/cli/templates/fragments/vue/App.vue.hbs` is the source for generated Vue app shell output.
- `apps/examples/README.md` documents the generated examples workflow.

## Forbidden Patterns

- Do not add hand-written React or Vue source under `apps/examples`.
- Do not commit `.generated/` projects as fixtures.
- Do not update examples without corresponding CLI template/runtime changes.
