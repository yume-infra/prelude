# Directory Structure

> How generated frontend scaffold code is organized in `create-yume`.

---

## Overview

This package does not contain a running frontend app. Frontend guidance here describes the React/Vue projects generated by the CLI and the template/registry code that produces them.

User-visible frontend output starts in `apps/cli/templates/`; runtime registration and conditional inclusion live in `apps/cli/src/core/template-registry/` and `apps/cli/src/core/owners/`.

## Directory Layout

```text
apps/cli/templates/
  fragments/
    common/          README, index.html, CSS, TS config, Vite, ESLint
    react/           React pages, components, router, stores, main
    vue/             Vue SFCs, router, stores, main
  partials/
    global/          shared partials
    react/           React JSX partials
    vue/             Vue SFC partials
  assets/            static copied assets

apps/cli/src/core/template-registry/
  frontend-app.ts    shared frontend template assembly
  react.ts           React family registry
  vue.ts             Vue family registry
  workspace-bootstrap.ts

apps/cli/src/core/owners/
  router.ts
  state-management.ts
```

## Generated Project Shape

- React output uses `src/main.tsx`, `src/pages/`, `src/components/`, `src/router/`, and `src/stores/` when options require them.
- Vue output uses `src/main.ts`, `src/views/`, `src/router/`, and `src/stores/` when options require them.
- Shared frontend output includes `index.html`, Vite config, TypeScript config, root CSS, README, lint config, and optional code-quality files.

## Examples

- `apps/cli/src/core/template-registry/frontend-app.ts` assembles shared core, lint/git bootstrap, finishing templates, code-quality templates, and family-local templates in a stable order.
- `apps/cli/src/core/template-registry/react.ts` maps React templates to generated paths and TS/JS extensions.
- `apps/cli/src/core/template-registry/vue.ts` maps Vue templates to generated SFC/view/router/store paths.
- `apps/cli/templates/fragments/react/App.tsx.hbs` and `apps/cli/templates/fragments/vue/App.vue.hbs` show framework-specific app shells.

## Forbidden Patterns

- Do not create a separate frontend app under `apps/cli/src`.
- Do not hand-edit generated output under `apps/examples/.generated`; change templates and registries.
- Do not put React-only or Vue-only semantics into shared frontend config.
- Do not use template helpers that are not registered in `apps/cli/src/core/services/template-helpers.ts` and documented in `docs/agent/handlebars-helpers.md`.
